W3C-DOM Öncesi Yöntemler

Genel Bilgiler

W3C Belge Nesne Modeli (W3C Document Object Model) veya kısaca W3C-DOM 1.0 spesifikasyonu 1998 de açıklanmadan evvel web sayfalarının ,stil sayfaları(CSS) ve JavaScript kullanımı ile hareketlilik kazanmasına Dinmaik HTML (DHTML) adı verilmekteydi. DHTML standart bir yöntem olmayıp, belge çözümleyicilerine özel yöntemler olarak gelişmiştir. Bunun sonucunda da, DHTML yöntemleri genel olarak ortak destekten yoksun web sayfalarına etkileşimleri sınırlı yöntemler olarak nitelendirilebilirler. Buna rağmen W3C-DOM öncesinde, çok sayıda script yazılmış, JavaScript örnekleri yapılmış, ticari programlar geliştirilmiştir. Bu programların anlaşılması, günümüz teknolojine çevrilmesi için, bu konuda bilgili olmak gereklidir. Burada, bu yöntemlerin kısa olarak tanıtılması yapılacaktır.

Web Sayfalarına Yazma

write Metodu

write Metodu, belirtilen pencereye bir veya birkaç JavaScript ifadesi yazdırır. writeln Metodu ile tek farkı satır sonuna satır sonlandırıcısı eklememesidir. Satır sonlandırıcısı ise, zaten HTML kodlarında geçersiz olduğundan, HTML sayfalarında write ile writeln( Metodları, etki açısından birbirinden farksız sonuçlar verir. Sözdizimi, 

write(ifade1 [, ifade2], .[, ifadeN])

şeklindedir.

Argümanlar

ifade1,ifade2,ifade3... : JavaScript ifadeleri. (İsteğe Bağlı)

writeln Metodu

writeln Metodu, belirtilen pencereye bir veya birkaç JavaScript ifadesi yazdırır. write Metodu ile tek farkı satır sonuna satır sonlandırıcısı eklemesidir. Satır sonlandırıcısı ise, zaten HTML kodlarında geçersiz olduğundan, HTML sayfalarında write ile writeln( Metodları, etki açısından birbirinden farksız sonuçlar verir. Sözdizimi, 

writeln(ifade1 [, ifade2], .[, ifadeN])

şeklindedir.

Argümanlar

ifade1,ifade2,ifade3... : JavaScript ifadeleri. (İsteğe Bağlı)

write ve writeln Metotlarının Uygulanması

Genel olarak web sayfalarında aynı sonuç verdiklerinden, uygulamada daha çok writeln metodu kullanılır. writeln Metodu, web sayfasının <head> veya <body> kısmındaki bir <script> elementinin içeriği olarak kullanılabilir. İstendiğinde, writeln metodu *.js şeklinde bir dış script dosyasında, bir JavaScript program adımı olarak da yazılabilir. writeln Metodunu içeriğinin sayfanın neresine yazacağı belli olmaz. Sayfanın yüklenmesi devam ederken, sayfanın herhangibir elementinden sonra, sayfanın yüklenmesi tamamlanmışsa, sayfa penceresi kapanır ve yeni açılan bir pencereye write metodunun içeriği yazılır.

Yeni bir boş pencereye bilgi yazılmak istenirse, yazma adımının bulunduğu fonksiyonu, <body onload="verileriYaz();"> şeklinde bildirilebilir. Yazma sayfanın belirli bir yerine yapılmak istenirse, bir fonksiyon kullanılmaz, yazılar bir <script> elementinin içeriği olarak, yazılmanın yapılması istenen yerdeki elementin altına yerleştirilir. Programcının şansı varsa, yazmak istediği bilgiler, yazmak istediği yere yazılır.

document.writeln() Metodu, bir sonucu olan her türlü ifadeyi yazdırabilir. Ayrıca, karakter dizigisi olarak yazrılıması istenen veriler, (X)HTML elenetlerini de içerebilir. Bu şekilde, istenirse, tüm bir web sayfası yazılabilir. Buna rağmen, böyle bir işleve gerek olmayabilir. Modern düşünce ile , sayfanın belge kodlarının (X)HTML, Görüntü kodlarının CSS, program kodlarının JavaSript ile yazılması hedeflenmektedir. JavaScript ile içerik yazmanın bir anlamı olmayabilir.

Aşağıdaki örnek, bu satırın hemen altına "Merhaba Dünya !" karakter dizgisini yazdıracaktır. 

<script type="text/javascript">
document.writeln("<p class='cursive-green'>Merhaba Dünya !</p>");
</script>

Bu şekilde bir yazma sistemi, kolay olmasına karşın, sayfanın kod sistemini düzensiz <script> elementleri ile doldurduğu için, sayfa kod düzenini bozucu ve kontrolünü zorlaştırıcı özelliğinden dolayı terkedilmesi sağlık verilmektedir.
