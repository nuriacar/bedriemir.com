3.6 - Classes

Classes are the abstract definitions of the objects, they definitions consist of properties. The properties of a class may be variables or functions which are called methods in Java. Class is a closed set and the properties of a class is completely insulated from outside world in Java.

A fundemental layout of a class may be as follows: 

ClassName{
[fields]
[methods]
}

Fields are variables whose types may be any calculable type of Java. They can be primitive types as well as predfined or user defined data types that means class structures. Methods as we have seen, are functions embedded in class definitions.

Fields and methods of a class may be,

  • static
  • non-static
We will begin by the introducing static fields and metods.

3.6.1 - Static Fields and Methods

Static fields and method are labelled with a keyword static. They belong to the class and invoked by calling the class definition itself. They are like the global variables and hanging functions of the functional programming languages like JavaScript.

When a class whose definition includes static properties (either fields or methods) is compiled, his static fields become utilisable from all the other classes in the same pckage and from the classes of other packages by importation. As an example a class whose name is Arithmetical and whose structure includes static properties is presented below: 

package objects;

class Arithmetical {
static double multiply ( firstElement , secondElement) {
return firstElement *  secondElement;
}
}

This class is neither compilable, nor referenceable outside the borders of the package that it is stored. Well, we did not mentioned anything preventing him to not to do so, but if you do not say anything, the compiler understands what was defaulted for. This is the definition of a class with default access permissions and default access permisson restrict the access of a class only within the package which is declared. We will cover soon the extra access permissions.

A class may be,

  • Executable (Compilable)
  • Non-Executable (Non-Compilable)

A class is the only entity that include a Java program. Java program listings or with the other name, compilation units, compiled to bytecodes and stored in a class files. The filenames of the class files must conform the class name case sensitively. All classes are stored in a separate folder, named after the package name which includes the classes ,again case sensitively.

An executable (compilable) class, include a main() method in his declaration. We have used these kind of classes in all over our examples so far and we are absolutely familiar with it. A compilable class is the only entity that include a Java program. A file may contain many Java classes, but only one of them may include a main() method. This class, is the executable class and the file must bear his name only.

A non-executable (non-compilable) class, does not include a main() method in his declaration. We are encountering one example right now. The Aritmetical class has no main() method, but it is stored in the folder of his indicated package. According the permission given in his definition for invocation, other classes may access this class. What is actually meanining of access? Accessing a class means to be able to produce intances of this class, calling his methods according their permission granted, getting and setting the values of the class members open to the public. All may be done according the restrictions and grants determined in the definition of the class.

An instance of the class is an object, structured according the recipe given in the definition of the class. An instance of the class is generated by calling a special method called the constructor of the class. If a class does not have a constructor, the default constructor is invoked. The default constructor is the class name followed by empty parenthesis, just like the one shown below: 


Arithmetical dt =new Arithmetical();

Interesting, the type of the variable dt is the class whose instance is created. This means the created instance is of a new data type. That is true, a class is a collection of members of different data types, including other classes. But this not all, a class also encapsulates method acting upon these class members. This is the new and important behavior. A Pascal record is new data type but is only assembles various componenets of various data types no methods. So is C++ structs and Ada's records. But the class structure is unsuspiciously superior to all of them. By including methods acting upon class members, Java classes have the ability to mimic many of the natures living and non-living formations. For example stellar motions may be successifully simulated by Java classes. OOP was known long before Java, but Java is designed specially for OOP and that is why Java classes are more successiful to handle natural events.

Since in the definition of Arithmetical class no constructor function is defined, default constructor is used when generating a new object instance from the Arithmetical class definition.

However, for accessing static fields or methods no instance generation is necessary. We simply access the static property by puuting a dot after the class name. A new class is designed for illustration of how a static field of class is accessed within another class: 

package objects;
class ArithmeticalAccess;
public static void main(String[] args){
double r = Aritmetical.multiply(25.0 , 10, 0);
System.out.println(r);
 } // end main
} //end class ArithmeticalAccess

The result of this program : 250.0

So far, this is very simple to use static fields. They are like standalone premade functions of any functional language.

Non-static fields are called class fields for class members. Class members are the blueprint of the structure of the class instances. All the instances generated from the same class have all the same structure i.e. same fields and same methods. Form one class we can produce billiions of instances having all the same structure but they may have different internal state i.e. the values of their fields may be different.

Access permissions are very important both for the classes and their structure. We will first introduce how to restict accesses and try some examples.
