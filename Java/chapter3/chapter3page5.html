3.6.2 - Access Modifiers (Continuing);

Access modifiers (or access specifiers) used with class members are as follows:

  • public
  • private
  • protected
  • No access modifier (default)
  • static
  • final
  • transient
  • volatile

A Java class ressembles to a formation like a coconut. It may be a completlely closed structure. But from a tightly closed coconut you can not sip the tasteful juice. A Java class is also the same, if you label all your class memebers as private, you will cut all the communication possibility between the outside world and your class, so nothing in, nothing out and the class will be completely useless in terms of useful material. A general rule of thumb is, make data field private but accessible by two public methods, one getting the values of the class members, this is called getter methods, the other one is to supply values to the class members these are called setter methods. It is strictly not advisable to let your class members altered by the program steps of other classes.

Non-static fields and methods are the blueprints, describing how the structure of the instance objects, generated from this class definition will formalize. The class structure mandates the structure of all the instances of this class. The names of the instances should be different from each other, because altough the structure of all the instances generated from the same class is strictly the same, their internal state may be different. Internal state means the values of the class members.

New instances of class are produced by calling the class constructor by a new paradigm.

Each class is a new data type. It is the agglomeration of the chosen existing data type including other defined classes. It may be simple by some primitive type class members and simple methods acting upon them. Or it maybe utterly comples structure with data memebers as classes themselves which includes data members of other classes and with internal classes. Consider the System class of the default java.lang package. This class has a class member named out which is itself a class and includes a variety of methods including println methods that we are using all the times to display our results.This is public static method so we call easily by System.out.println(), without the need to pass from the intermediate System class instance. This shows that a class structure may be huge and complex. To sort it out, knowledge is essential.

Class members specified as final, are merely constants. Their values may be once set in their definition and can not be changed in the subsequent steps of the program.

Static data members, as we know before, are not belonging to class instance structure. They are belonging to the class structure and they can only be called as ClassName.fieldName paradigm.

Protected data members may only be accesible from the class itself and his hierarchical descendends made by extending the class. They are useful in inheritance.

Static fields are common to all objects. These fields are not related to the class instances but to the class itself, that is the reason they are called class variables or static fields. Static fields are accessed and can be changed by any object instance, but their access need not producing class instances, they can simply be called as ClassName.classVariable (static field)

Transient and volatile class members will be introduced later with Java threads.

Class methods acts uopn the class members, they synthax is as follows

[Access Modifiers] [Usage Modifiers] return-type methodName (formal-parameter-list) (exception-list)

Access Modifiers(Access specifiers) used with methods are as follows: (only one of them may be applied)

  • public
  • private
  • protected
  • Not Mentioned (Default)

Public methods are accessible from every class, from everywhere. In order to use a public method decşlared in some other package, this package must be imported or the metod should be called with his fully qualified name. that is: packageName.ClassName.methodName(actual-arguments-list)

Private methods are for internal use of the class. They are only available within the class only. These methods may not be abstract methods.

Protected methods are available to the class and his subclasses. They are useful in inheritance.

If no access modifier are pronounced by the definition of the method, the method is supposed to be restricted for to be accessed only from within the package where the class is declared. This is the default behaviour of any method. There is no reason to restrict the access of a method only to his originating package.

Usage modifiers used with methods (all of them may be used together)

  • static
  • final
  • abstract
  • native
  • synchronised

Abstract classes contain at least one abstract abstract metod.An abstract method, contain only definitions which will be implemented later.
