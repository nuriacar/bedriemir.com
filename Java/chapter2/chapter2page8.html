2.6.1.1 - Assignment Operator (continuing)

Assigning a value to a variable, is a way to determine the state of the system. The state of the system is defined as position when all the variables are at some determined value. When we assign some value to a variable we bring the system to certain state. When a system is at some defined state, assigning some new value to any of the variables, will change the state of the system. Or in other words, will bring the system in another state.

In java, when declaring a variable, it is compulsory to give its initial value, otherwise an exception will be raised. Theoretically, "An exception occurence is synonymous with impossibility of delivering the standard service specified for a program." as described by Flaviu Christian. Practically, it is synonymous of a program error. In computing words, when an abnormal condition is encountered in a program, it is said that an exception is raised, or an exception is thrown. If an exception is thrown while in compiling, it is called a compile time exception. If it occurs while running the program, it is called a runtime exception. In Java, exceptions may be handled, thus preventing the abrupt ending of the program when an exception is thrown. If an exception is thrown and it is not properly handled, the error will propagate to the higher areas of the program and most of the time the program will end abrubtly. We say that the program is crashed and it may also trigger the events that may crash the operating system. In Java, when there is a detectable error in the program, Java compiler stops compiling and issues a message indicating why the compilation was not succesiful. If an exception is thrown in the runtime, program execution stops and a runtime exception message is generated.

Let us see, what will happen if we force the compilation of a program where there is a declaration of the variable, without an initial value: 

package preliminary;

public class Uninitialized {

public static void main(String[] args){
 String name;
System.out.println(" Name = " + name);
 } // end main

} //end class Uninitialized

The result of this program : 

      Exception in thread "main" java.lang.Error: Unresolved compilation problem:
  The local variable name may not have been initialized
  at preliminary.Uninitialized.main(Uninitialized.java:8)

This is compile-time exception. Compile-time exceptions are generated when there is an illegal statement in some program syntax. They are also called coding errors.

Perhaps here a liitle warning may be helpful. If a program runs smoothly without any exception raised, that is not a sign that this program is running without error. There might be some logical errors in program design, that will produce wrong results or some other issues that are not raising any exception but turns the program to produce wrong results. All the programs should be fully tested and only delivered to the industry when there is a full consent that they are producing reliable results.

Returning to our example, the expression "Name : " + name, is a string concatenation. The meaning of the string concatenation is the seamless integration of two String type data. In computer speaking, string concatenation is a contigious juxtaposition of two String type data. Contigious means there is no space between previous and next data.

String concatenation may also convert from numerical data types to String data type. For example to convert 306 (When we do not supply any suffix the compiler defaults to int for integer literals) to String, just add an empty string like "" + 306, the result is "306".
