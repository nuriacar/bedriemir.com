2.6.1.1 - Assignment Operator (continuing)

Let us experience with some literals: 

package preliminary;

public class Literals{

public static void main (String[] args) {

double length = 45.77D;

// Suffix D is not necessary and it is always omitted !

float width = 12.66F;

// /* Do not use floats, but if you use them suffix F or f 
should always be stated. Otherwise 
the value will be understood as of type double ! */

double length = 45.77;

Long population = 25_888_346_663L;
/* Do not use long integers unless there is an absolute need.
But when we use them the suffix L (preferably not l) 
should always be stated. Otherwise, 
the value will be understood as of type int ! */

// Note that the underscores are only supported by JVM SE 7 or later !

int binaryCode = 0b1001001;
*Binary literals are only supported by JVM'S SE 7 or later!
After the input, JVM convert them to the euivalent
value in base10 integers, 
They are used and output with their converted values ! */

int HexCode = 0x7FFA;
/*Hex literals are  treated like binary literals.
After the input, JVM convert them to the euivalent
value in base10 integers, 
They are used and output with their converted values ! */

System.out.println("binaryCode (decimal equivalent) :  \t"  + binaryCode);

System.out.println("HexCode (decimal equivalent) :  \t" + HexCode);

System.out.println("Length :  \t" + length);

System.out.println("Width :  \t" + width);

System.out.println("Data :  \t" + data);

System.out.println("Population :  \t" + population);
}
}

The output of this program: 

binaryCode (decimal equivalent) :   73
HexCode (decimal equivalent) :    32762
Length :    45.77
Width :   12.66
Data :    45.77
Population :    25888346663

When a String data is concatenated with a numeric data like

"Width : \t" + width;

The result is a new String comprising from the String + value of the numerical data converted to String data type. It follows that the result of this concatenation is a data of type String and the compiler will call the method println(String x).

Note that there is a tab character in the string message as

"Width : \t"

and this make the ongoing concatenation, equivalent to:

"Width : " + "\t" + width;

This is the reason why the value of the variable width is displayed one tab length apart from the message as:

Width : 12.66

In the other hand, pay attention that the message can be anything like

"Width : \t"

but the numerical data to be concatenated with a String data must match a defined variable case sensitively. If we state "Concat" + DataValue, without defining DataValue, compiler will reject the concatenation since DataValue is not defined in the program. Meanwhile, "Concat" + data will result in a new string "Concat45.77", because data is a defined variable of the program.
