2.6.1.1 - Assignment Operator (continuing)

Java is a block structured language. Blocks are delimited by curly parenthesis {}. Anything declared within a block, is only accessible within this block only and completely invisible from outside of this block. This brings ease of programming, extra modularity and data safety in Java.

Consider the main() method, 

public static void main(String[] args){

 } // end main

There is only two curly brackets, one opening and one closing. This is called a definition block. Anything declared in the main() method, is declared with the definition block of main() and accessible only from the elements defined in the definition block of main(). For the present let us focus on the variables declared in the main() method. Any variable declared in the main method is only accesible from the definition block of main(). This is called visibility of the variable. Any variable is only visible within the block which it is defined.

Nothing prevents us to to place more definition blocks in a main() method. In this case, any block, defined within another block is an inner block. We can can define as many inner blocks within a block and with any depth. But deeper inner blocks are not advisable, a programmer may easily loose control with deeper inner blocks.

If we open two inner blocks in the main() method, we will have a program structure displayed below:

public static void main(String[] args){ // beginning of a definition area of main
{ // beginning of the inner block A

} //end of the inner block A

{// beginning of the inner block B

}// end of the inner block B

 } // end of the definition area of main

Note that, nothing is changed in terms of the definition area of main(). Anything declared in main(), is still visible up the end of the definition area of main(), except within new opened definition blocks A and B. The definition block of of main() contains the definition blocks A and B therefore, the definition block of main() is the hierarchical parent of A and B. A and B are child blocks of their common parent block main(). When child blocks share a common parent, they are named siblings of each other. The blocks A and B are siblings. This is a small introduction to the hierarchical system which is the founding concept of the Java programming language.

The variables declared in main(), are only visible within main() and his chidren. Within the sibling blocks, only the variables defined in main(), and whose defined within this sibling block are visible. Any variable defined in a sibling block is neither visible from within another sibling block nor from the parent block.

From these definitions we can conclude:

  • Any variable declared in main is visible from A and from B
  • Any variable declared in A is not visible from B
  • Any variable declared in B is not visible from A
  • Any variable declared in A is not visible from main()
  • Any variable declared in B is not visible from main()

The hierarchical based scope and visibiliy rules of Java is very simple, very straightforward. very intuitive, nothing to memorize. It may be better understood after drawing some simple sketches of the program structure and colorize different visibility areas with different colors.

Generally the variables are defined in the beginning of blocks, but when it may induce more clarity, they may may also defined inside the definition block. But it is better to keep the definitions of all the variables assembled and in the beginning of the definition block, otherwise there is risk of overlook this definition in the debugging and maintenance stages of the program.

A program below, may help to clarify the scope and visibility of the variables in Java.

package preliminary;

public class Lifetime1 {

public static void main(String[] args){
{ // beginning of a definition area I
 int time = 2;
System.out.println(time);
} //end of a definition area I

{// beginning of a definition area II
System.out.println(time);
}// end of a definition area II

 } // end main

} //end class Lifetime1

The results of this program: 

Exception in thread "main" java.lang.Error: 
at preliminary.Lifetime1.main(Lifetime1.java:12)
Unresolved compilation problem: 
time cannot be resolved to a variable

The independence of the definition blocks may be clearly understood from the program above. The compiler rejects the the compilation because there is an undeclared variable in the second block. This variable was defined in the first block but it is invisible from inside the second block. This is why the compiler has rejected the compilation of this program.

If we respect the independence of the definition blocks and abide with the visibility rules, the program will compile without any problem, just like the one below: 

package preliminary;

public class Lifetime2 {

public static void main(String[] args){
{ // beginning of a definition area I
int time = 2;
System.out.println(" Time = " + time + " Hours. ");
} //end of a definition area I

{// beginning of a definition area II
int time = 2;
System.out.println(" Time = " + time + " Hours. ");
}// end of a definition area II

 } // end main

} //end class Lifetime2

The result of this program : 

      Time = 2 hours.
      Time = 2 hours.

If there is not an absolute need, there is no reason to define subdefinition areas (inner blocks) in a definition area. Most methods have only one definition area. In our example above, there is also not any need for declaring subdefinition areas. We might simply write as shown below: 

package preliminary;

public class Lifetime3 {

public static void main(String[] args){
 int time = 2;
System.out.println(" Time = " + time + " Hours. ");
 } // end main

} //end class Lifetime3

The result of this program : 

      Time = 2 hours.

But, sometimes there might be a need to define sub definition areas like in the program below: 

package preliminary;

public class Lifetime4 {

public static void main(String[] args){
{ // beginning of a definition area I
int time = 2;
System.out.println(" Time = " + time + " Hours. ");
} //end of a definition area I

{// beginning of a definition area II
int time = 3;
System.out.println(" Time = " + time + " Hours. ");
}// end of a definition area II

 } // end main

} //end class Lifetime4

The result of this program : 

      Time = 2 hours.
      Time = 3 hours.

In the class Lifetime4(), the value of variable time is different in different definition areas and therefore the insertion of two child blocks became necessary.
