2.6.1.1 - Assignment Operator (continuing)

After having a single string data in String data type, we need a suitable means to display it. As we have mentioned at section 1.4.1, java.lang package, has a System class with a member named out from the printsteam() class and this particular printsteam() class has a print(String x)/ println(String x) method to display String type data. That is a rescue but for only String data type. What we will do if we have some int type result to display? Since we are provided with a build-in method that accepts String type argument, we can convert our int type data to String data type by concatenation with an empty string and call the method print(String x) or println(String x). But every time we intend to display numerical data, the necessity to cast (data type conversion) to a String type, is annoying and time consuming. We we'll welcome a better choice. And Java provide us what we need in a golden plate.

In Java, a class may have as many methods to suit the needs. Some methods may have the same name but with different arguments. Online documentation show that the class printstream() has a multitude of print()/println() methods with different argument types. This is called method overloading in Java and it is amongst the most useful features of Java.

Overloaded methods have the same name, but different arguments. When an overloaded method is called with a specific argument, Java compiler analyse the argument and loads the appropriate method. This is one of most valuable help from the compiler and it is one of the superiority of Java among his competitors. For example C++ does not support method overloading.

Method overloading is one the topics of the Object Oriented Programming (OOP) and we will fully introduce and experince with overloaded methods when we will study OOP. Here, we have introduced method overloading, because we are applying it in our preliminary examples.

You may call freely overloaded methods like print()/println(). Java compiler will find the suitable print()/println() method to display your data without any difficulty.

When a value is assigned to a variable, for example with specifying an initial value, subsequent assignment of a new value to this variable, cause the loss of the old value, which is no longer accessible. This is called a destructive assignment. An an example of desructive assignment is given below: 

package preliminary;

public class DestructiveAssignment {

public static void main(String[] args){
 int NumberOfParticipants = 2;
NumberOfParticipants = 3;//Destructive Assignment!!!
System.out.println(" NumberOfParticipants = " + NumberOfParticipants);
 } // end main

} //end class DestructiveAssignment

The result of this program : 

      NumberOfParticipants = 3

In the program above, a new value is assigned to a variable NumberOfParticipants. With this new assignment the old value of the variable is lost and no longer accessible. The variable has a new value. The system has passed to a new and different state. The previous state of the system is unrecoverable. This is an example of destructive assignment.

If we intend to preserve existing value of variable, the only thing to do is to declare a new variable and assign the new value to this new variable. With that, we may access the old and new values for an entity. This is called non-destructive assignment. One example is given below: 

package preliminary;

public class NonDestructiveAssignment {

public static void main(String[] args){
 int NumberOfParticipants = 2;
int ActualParticipants = 3;//NonDestructive Assignment!!!
System.out.println(" Initial Number Of Participants = " + 
NumberOfParticipants);
System.out.println(" Actual Number Of Participants = " +
ActualParticipants);
 } // end main

} //end class NonDestructiveAssignment

The result of this program : 

      Initial Number Of Participants = 2
      Actual Number Of Participants = 3

Non-destructive assignment provides us with the previous value of a variable but we should assign one more memory location than the destructive assignment. For bigger programs, especially with working huge amount of data, constantly allocation of new memory locations to the variables for the sake of non-destructive assignments, may cause a shortage in system resources. Knowing that, do not apply non-destructive assignments, when there is no an absolute need.
