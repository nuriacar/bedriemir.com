Sayılabilir Veri Tipleri İçin Öntanımlı Giriş/Çıkış Prosedürleri

(Standart Giriş/Çıkış Ortamı Uygulamaları)

Ek-3 Sayfa 4

Ada.Text_IO Öntanımlı Paketinde Sayılabilir Veri Tiplerinin Giriş/Çıkış Prosedürlerinin Tanıtımı

Ada 2012 spesifikasyonu Annex AQ.10.1 deki generik Ada.Text_IO_Enumeration_IO paketindeki, Sayılabilir (Enumeration) tipte veriler için, giriş/çıkış prosedürleri, aşağıda görüldüğü gibi, öntanımlı olarak verilmiştir:

-- Generic package for Input-Output of Enumeration Types
generic
type Enum is (<>);
package Enumeration_IO is

Default_Width : Field := 0;

Default_Setting : Type_Set := Upper_Case;

procedure Get(File : in File_Type;

Item : out Enum);

procedure Get(Item : out Enum);

procedure Put(File : in File_Type;

Item : in Enum;

Width : in Field := Default_Width;

Set : in Type_Set := Default_Setting);

procedure Put(Item : in Enum;

Width : in Field := Default_Width;

set : in Type_Set := Default_Setting);

procedure Get(From : in String;

Item : out Enum;

Last : out Positive);

procedure Put(To : out String;

Item : in Enum;

Set : in Type_Set := Default_Setting);

end Enumeration_IO;

Bu aşamada, sadece standart giriş/çıkış (ekran) giriş/çıkış prosedürlerini uygulayacağız. Ekran giriş/çıkış prosedürleri, giriş için Get, çıkış için Put prosedürleridir. Aşağıdaki uygulama ile bu giriş çıkışların nasıl çalıştığını gösterelim : 

with Ada.Text_IO; 
procedure e3_s4_uyg_1 is
type Renkler is (KAHVERENGİ, KIRMIZI, YEŞİL, SARI, TURUNCU, MAVİ, PORTAKAL, HAKİ, BEYAZ, SİYAH);
package Enum_IO is new Ada.Text_IO.Enumeration_IO (Renkler);
Renk : Renkler;
begin
Ada.Text_IO.Put(Item => "Lütfen Renkler Kategorisinde Tanımlı Renklerden Birini Giriniz !");
Ada.Text_IO.New_Line;
Enum_IO.Get(Item => Renk);
Ada.Text_IO.Put(Item => "Girilen Renk : ");
Enum_IO.Put(Item => Renk , set => Ada.Text_IO.Type_Set'First);
end e3_s4_uyg_1;

Bu programın sonucu,

SARI
Girilen Renk : sari

olmaktadır.

Program ilk olarak Renkler sayılabilir veri tipinin tanıtımı ile başlamaktadır. Bunan sonra, Ada.Text_IO.Enumeration_IO generik sayılabilir veri giriş/çıkış öntanımlı paketinin, Renkler veri tipi özel anlık örneklemesi (instantaniation) yapılmakta ve Enum_IO non-generik giriş/çıkış paketi tanımlanmaktadır. Son olarak Renk adında, Renkler veri tipinden bir değişken tanımlanmaktadır.

Program başlayınca, lilk olarak Renk değişkeni için konsoldan bir değer okutulmaktadır. Renk değişkeni, Renkler olarak tanımlanmış sayılabilir veri tipindedir. Renkler veri tipi, tüm sayılabilir veri tipleri gibi alttan ve üstten sınırlı ayrık, skaler bir veri tipidir. Bu veri tipinden bir değişkene yasal olarak sadece tanım aralığında bulunan bir değer atanabilir, Aksi halde hata oluşur. Bundan sonra okununan değer, Renkler veri tipi için spesifik Enum_IO paketinden yararlanılarak görüntülenmektedir. Bu paketin Put prosedürünün Type_Set argümanı, Ada.Text_IO program paketinde tanımlıdır. Program başlangıcında, Ada.Text_Io dan yararlanılacağı with Ada.Text_IO; bildirimi ile bildirilmiş fakat bu paketin içinden çalışılacağı use Ada.Text_IO; bildirimi ile yapılmadığından, ADA.Text_IO öntanımlı paketinde tanımlı olan Type_Set değişkenini, tanımlı olduğu ve bu programın kullanımına açık olan Ada.Text_IO.Type_Set'First şeklindeki tam nitelikli (Fully Qualifed) şekliyle belirtilmiştir. Eğer use Ada.Text_IO; bildirimi yapılmış olsaydı, paketin içinden çalışır gibi olacağımızdan, Ada.Text_IO öntanımlı giriş/çıkış paketinde tanımlı olan, Type_Set sayılabilir veri tipi görünür olacak ve çağırmak için tam nitelendirmeğe gerek olmayacaktı.

Yukarıdaki programda olduğu gibi, bir prosedürün birlikte çalışıp, içiçe çalışmadığı bir paketin metotlarını çağırırken, tam nitelikli isimlerin kullanımı, biraz uzun ve belki de göze hoş görünmeyebilecek program satırlarına neden olabilir.Eğer use bilidirimi de yapılsa idi, bu satır, 

with Ada.Text_IO; use Ada.Text_IO;

procedure e3_s4_uyg_1 is

Enum_IO.Put(Item => Renk , set =>Type_Set'First);

end e3_s4_uyg_1;

olacaktı. Bu çözüm ile tam nitelikli isimlerin kullanımına gerek olmamakta fakat, tüm paket için use bildirimi, isim uzayı kirlenmesine neden olabileceğından sağlık verilmemektedir. Prosedürlerde, paketler için use bildiriminin aşırı kullanımı isim uzayı (namespace) kirlenmesine yol açar ve sağlık verilmez. Burada anahtar sözcük, aşırı sözcüğüdür. Eğer, birden fazla paket use bildirimi ile içinden çalışılmaya açılırsa, bir tür büyük bir program alanı açılmış olur. Bu paketlerde aynı adı taşıyan metotlar varsa, derleyici, hangi paketin metodunun çağrıldığını, metodun argümanlarını inceleyerek anlamaya çalışacaktır. Bu da zaman ve uğraşı anlamına gelmektedir. Eğer derleyici metodun hangi paketten gelmesi gerektiğini ayırt edemez ise, bunu çelişki sayarak programı hata mesajı ile kesecektir. Eğer içinde çağrılacak paketlerde aynı isimli metotlar veya alt paketler yoksa, bu sakınca olmayacaktır. Yine de burada oldğu gibi, en küçük bir birliktelik olayında bie isim çakışmaları yaşanabilir. Örnek olarak, burada hem Ada.Text_IO hem de Enum_IO paketinde Get ve Put metotları bulunmaktadır. Burada use bildirimi kullanılırsa isim çakışması yaşanacağı açıktır. Aslında bu gibi küçük ve açık olaylarda, derleyici için bu çakışma hiçbir sakınca yaratmayabilir. Yine de, derleyicinin rahat çalışması için bu gibi, isim uzayı kirlenmelerine yol açmamak gerekir. O zaman öyle bir çözüm bulunmalı ki, hem çağırma için tam nitelikli isimler kullanımına gerek olmasın, hem de, isim uzayı kirlenmesi oluşmasın. Bu biraz sıradışı çözüm aşağıdaki literatürde önerilmiştir. Bu yöntem aşağıdaki uygulamada görülmektedir: 

with Ada.Text_IO; 
procedure e3_s4_uyg_2 is
type Renkler is (kahverengı, kirmizi, yeşıl, sari, turuncu, mavı, portakal, hakı, beyaz, sıyah);
package Enum_IO is new Ada.Text_IO.Enumeration_IO (Renkler);
Renk : Renkler;
Büyük_Harf : Ada.Text_IO.Type_Set := Ada.Text_IO.Type_Set'Last;
use type Ada.Text_IO.Type_Set;
begin
Ada.Text_IO.Put(Item => "Lütfen Renkler Kategorisinde Tanımlı Renklerden Birini Giriniz (Lütfen Sadece Büyük Harfler Kullanınız !)  :   ");
Ada.Text_IO.New_Line;
Enum_IO.Get(Item => Renk);
Ada.Text_IO.Put(Item => "Girilen Renk : ");
Enum_IO.Put(Item => Renk , set = Büyük_Harf;
end e3_s4_uyg_2;

Bu programın sonucu,

Lütfen Renkler Kategorisinde Tanımlı Renklerden Birini Giriniz (Lütfen Sadece Büyük Harfler Kullanınız !) : KIRMIZI

Girilen Renk : KIRMIZI

olmaktadır.

Program içinde, Büyük harflerle tanıtılmış olan SARI elemanı, Türkçenin Ada derleyicisinde Büyük harf/ Küçük Harf değişiminin yanlış gerçekleştirildiği i, ı, İ ve I harflerini içeren bir değer olduğundan, küçük harflerle yazdırılması sari gibi anlamsız bir sonuç vermiştir. Ada derleyicisi için, sari sarI, SARI, Sari vb yazılışlar aynı değişkeni işaret etmektedir. Bu nedenle, doğru yanıt alınması için iki yöntem uygulanabilir. Bunlardan ilki, ikinci programda olduğu gibi, bu harfleri içeren tanıtıcıların tanımlandıkları karakter büyüklüğü ile çağrılmaları yöntemidir. Bu yöntem, tanıtımda sıradışı bir karakter girişi gerektirmez fakat, elamanların doğru çağrılması için, mutlaka tanıtıldıklaraı karakter büyüklüğü ile çağrılmaları gerekir. İkinci yöntem daha esnektir fakat tanıtım sırasında biraz sıradışı karakterler kullanımını gerekir. Örnek olarak küçük harflerlerle çağrılacaksa mersin, büyük harflerle çağrılacaksa mersın yazmak gerekir. Yine aynı şekilde, küçük harflerlerle çağrılacaksa sarı, büyük harflerle çağrılacaksa sari yazmak gerekir.

Sayılabilir tipte veriler istenirse imaj nitelikleinden yararlanılarak, Ada.Text_IO öntanımlı paketinden yararlanılarak da görüntülenebilirler. Bu konuda bir uygulama, b4-s1.htm sayfasında görülebilir.
